# 个人求职简历（HPC开发岗位）

## 1 通用问题（非技术问题）
**考察点：从事HPC的动机和匹配度**
**重要问题：** 为什么从研发转到制造，又从制造跳到HPC开发？这2年存在技术“断层”。现在选择转行HPC是不是一时冲动？
**核心思想：** (1) 对技术有着强烈的兴趣和热情；(2) 同时2年看似无关的经验，锻炼了系统思维和工程级交付能力。

### 1.1 转行理由
**问题：** 为什么离职/转行，选择HPC开发岗的理由是什么？
**考察点：** 职业规划/动机的合理性，是否“一时冲动”。
**思路：**
1. 肯定过去：熟悉了开发工具和流程规范，独立完成了多个软件项目，培养了工程管理和系统思维。
2. 核心原因：**兴趣使然（驱动力）**。在探索业务方向时，发现了新的兴趣点，即高性能计算与优化，探索GPU架构、发掘底层硬件的潜力和并行计算的极限。
3. 实际行动（**重要**）：自学CUDA和HPC基础知识，并投入项目实践，已熟练掌握HPC基础。
4. 总结与展望：工作中积累的经验与能力，和对HPC相关知识的掌握，与岗位有着极高的契合度。

**范例：**
1. 过去的经验提供了扎实的积累，熟悉了研发的敏捷式开发流程(IPD)，掌握了业界常用的高效开发工具（Git、Linux等），也实现了多个完整的项目（从设计、开发到测试、交付），培养了扎实的工程交付能力、系统架构思维和高效的问题解决能力。
2. 在探索使用AI解决制造现场业务(智能TPM、设备部署、健康度等)时，发现部门数据分析系统提供的算子效率不理想、可优化，在逐步探索中逐渐喜欢上了GPU硬件与HPC知识体系。
3. 利用空余时间，自学并熟悉了GPU底层硬件、CUDA架构，独立完成了GEMM、并行归约算法等项目，并通过Nsight Compute进行性能诊断与针对性优化。在这一过程中，更加坚定了自己从事HPC的决心。
4. 综上所述，在3年多的工作中积累的软件开发能力，以及对软件项目管理的经验和思维，结合CUDA开发能力，正是这个岗位需要的。能够与团队协作，交付优化完备、高质量、可维护性强的的HPC工程代码，提升产品竞争力。

### 1.2 岗位稳定性
**问题：** 之前有2年项目管理经验，你确定会回到开发相关的工作吗，而不是转管理？
**考察点：** 岗位稳定性，是否愿意成为贡献者而非决策者，不是把技术路线当作跳板。
**思路：** 
1. 明确目标：职业目标就是成为技术专家。项目管理经验提供了一种更全面的视角，从业务价值的角度去看待。但如何高质量且高效地实现，是最关心的。
2. 技术热情：深入技术细节，制定合理可行的解决方案，并通过代码实现，完成高质量交付的技术路线，充满热情。
3. 转化优势：管理经验提供了系统思维，这是一种优势，能够更好地理解上层需求并转化为开发所需的语言，对高质量交付大有裨益。但技术上的实现还是个人最追求的。

### 1.3 证明自己能够快速上手HPC
**问题：** CUDA和HPC的学习曲线非常陡峭，而你没有相关的工作经验。如何证明你能够快速上手并产生价值？
**考察点：** 学习能力、自驱力、抗压能力
**思路：** 
1. 学习能力强：在研发和制造，独立完成并实现多个项目。首次接触自研芯片并设计与实现算法、SDK层软件设计，研究并熟悉整个项目的架构，快速上手C语言固件开发，实现性能提升xx。
2. 自驱力强：对CUDA和HPC有着极强的自驱力。利用业余时间学习GPU硬件、CUDA架构与开发，并实现GEMM、Reduction等多个项目的优化。内容全靠自学，足以证明抗压能力和自驱力。
3. 高绩效/产出：荣获《明日之星》个人奖，多次与团队拿下关键项目。
